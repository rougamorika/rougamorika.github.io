引入:什么是矩阵:一个半群?

所谓矩阵就是个数字按顺序排列的结果.每一个数可以用它所在的行和所在的列唯一地确定,我们称为行指标和列指标. 矩阵对加法和矩阵乘(就是普通的矩阵乘法,因为我们后面要讲不少奇奇怪怪的乘法)构成加法半群和乘法半群.以防有读者没学过半群(Semi-group),所谓的半群就是指一类满足 特殊条件的集合,条件如下:

  • 封闭性：集合中的元素对于给定的运算封闭,也就是集合内所有的元素经过给定的运算之后产生的元素仍在集合内
  • 结合律：集合中的元素对于给定的运算满足结合律

矩阵不管是加法还是普通矩阵乘都是满足这样的性质的,所以构成了加法半群和乘法半群. 我们之所以要这么来入门矩阵,是为了在研究加法和乘法的时候可以不假思索的使用结合律,其实这是果因倒置的,因为我们是先有的乘法,再定义它具有半群的性质.但是这能方便我们对矩阵的结构加以抽象,帮助我们跳出纯粹的计算,从一个代数结构的角度来看待它.

我想看看空间

线性代数是研究线性空间的艺术,那么我们就应当提出一个很典的问题:线性空间他妈的是啥?? 书上写得很无聊,满足8条公理,对加法乘法封闭,blah,blah…很烦.我们就取他是所有向量所构成的一个集,然后它满足向量的加法和数乘(也就是对他俩封闭). 那么问题在这里,矩阵在这样的空间中是个什么样的玩意??很多人看过了Gilbert老爷子讲解消元法了,那么大概也能知道矩阵是一种对线性变换的描述,你可以通过矩阵来描述一个,这个时候我们时常写 但是我们以一个更简单也更直接的视角来看待矩阵:好几个列向量的有序列. 事实上,由于初等行/列变换的存在,我们也不必要求这个列是有序的.所以矩阵简单说,就是一堆列向量(或者一堆行向量)构成的集合. 那么我们的加法就相当于在处理两个向量集合同一对应位置的和,向量的加和被定义为坐标的相加,所以矩阵的加和也就只能是对应元素的相加.

乘法

我为什么不能直接像加法一样? 矩阵的 Hadamard 积

这是所有线代人最直接的想法,因为加法有一个极其良好的性质,他不改变矩阵的大小,或者这么说,只有同大小的矩阵才有加法.也就是矩阵加法是保维度的. 我们自然也会想要构建一个保维度的乘法,我们只需要把两个同型矩阵对应指标的元素相乘得到乘积矩阵对应指标的元素即可,也就是逐项乘积,这种乘法当然存在,称之为矩阵的Hadamard 积,也叫做Schur积,分别命名自Jacques Hadamard和Issai Schur. 我们记: Hadamard积这么直观,为何我们不用Hadamard积呢?这是因为Hadamard积有一个巨大的漏洞:不保线性变换. 一个保加法(满足可加性)和保数乘(满足齐次性)的变换就称之为线性变换,我们现在来证明Hadamard积不保线性变换,假设我们用来表示Hadamard积造成的变换: 所以Hadamard乘积并不保数乘. 现在我们计算 ( (A B)( + ) )： [ (A B)( + ) =
